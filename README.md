## Overview
Antibiotic resistance is a global challenge, and plasmids are key mobile genetic elements involved in horizontal gene transfer.

This repository analyzes plasmid relationships from a **network perspective**, using **distance matrices generated by `pling`** as input.  
The core contribution here is the **igraph-based code** that transforms those distances into weighted graphs and studies their structure with centrality and community metrics. In the future, we may include other input types or formats, but for now, the focus is on leveraging `pling` outputs to explore plasmid evolution and organization.

---

## Scope
This project **does not run `pling`** internally.  
Instead, it assumes `pling` results are already available and focuses on:

- Building weighted plasmid graphs from `pling` distance output
- Computing graph metrics (degree, eigenvector, betweenness, closeness)
- Detecting structural roles inside and across communities
- Exploring possible backbone organization across plasmid families

---

## Research Question
To what extent can graph centrality metrics identify plasmids that act as organizing structures within communities, such as:
- core nodes,
- bridges between communities,
- and key structural elements in plasmid family organization?

---

## Hypothesis
Different centrality measures capture different structural roles:

- **Degree** and **eigenvector** centrality identify core or dominant nodes.
- **Betweenness** centrality identifies bridge nodes between communities.
- **Closeness** centrality identifies nodes with lower average path distance within communities.
- At a global level, highly central nodes may form a **backbone** linking major plasmid families.

---

## Methodological Pipeline
1. Load precomputed distance matrix files from `pling` output.
2. Convert distances into a weighted graph representation.
3. Analyze node-level centrality (degree, eigenvector, betweenness, closeness).
4. Detect community structure and bridge nodes.
5. Study global connectivity and potential backbone patterns.
6. Export rankings, summaries, and visualizations.

---

## Major Implementations

- `pling_igraph_viz.py`: builds attributed `igraph.Graph` objects from a
  community JSON + `typing.tsv`, computes a three-phase hierarchical
  layout (global FR → per-subcommunity KK/FR → centroid repulsion) and
  renders two PNG figures per community: the full graph and the no-hub
  graph.  The script requires a single `--out-dir` and always writes
  both `community_<n>.png` and `community_<n>_no_hubs.png` into it.

- `pling_igraph_analysis.py`: computes node-level centrality measures
  (degree, betweenness, closeness, eigenvector), clustering coefficients
  and path-length statistics; produces per-measure visualisations and a
  text summary (`summary_community_<n>.txt`) with raw metrics. The degree
  distribution summary also reports an igraph-based power-law fit via
  `igraph.statistics.power_law_fit`. Notes:
  - Closeness summary uses the Wasserman–Faust (WF) normalized value
    for disconnected graphs (recommended for validation/consistency).
  - Eigenvector centrality is computed only on the largest connected
    component; nodes in smaller components receive `0.0`.

All scripts keep vertex/edge attributes (name, color, is_hub, subcom;
sd/td/lbl) so you can import the graphs into other analysis pipelines.

## Installation

Recommended: use a conda environment (tested as `igraph_1`). Example:

```bash
conda create -n igraph_1 -c conda-forge python=3.10 python-igraph matplotlib pandas numpy
conda activate igraph_1
```

Alternatively install with `pip`:

```bash
pip install igraph matplotlib pandas numpy
```

## Usage

Render visualisations (both full and no-hub PNGs):

```bash
python3 pling_igraph_viz.py --json path/to/community_0.json \
    --typing path/to/typing.tsv --out-dir igraph_output_pling --dpi 200
```

Run full centrality analysis and produce PNGs + summary text:

```bash
python3 pling_igraph_analysis.py --json path/to/community_0.json \
    --typing path/to/typing.tsv --out-dir igraph_output_pling --community 0 --dpi 200
```

Outputs will be written under `--out-dir` (example `igraph_output_pling/`):

- `degree_centrality_full.png`, `degree_centrality_no_hubs.png`, etc.
- `degree_distribution_full.png`, `..._no_hubs.png` (degree distributions)
- `summary_community_0.txt` — raw numeric summary and top-K lists
- `path_length_histogram_*.png` (average path length / distribution)

## Notes and Recommendations

- Use the normalized metrics for plotting and for cross-graph comparisons;
  the summary file contains raw metrics for degree/betweenness/eigenvector
  but closeness is reported using the WF-normalized value due to
  disconnected components. This keeps validation consistent.
- Eigenvector centrality is restricted to the largest connected component
  to avoid spurious mixing across disconnected pieces.

## Quick Examples and Tips

- To re-run the example community used for testing (if present):

```bash
conda activate igraph_1

    --typing raw_pling_example_output/typing.tsv --out-dir igraph_output_pling --community 0 --dpi 200
```

## Expected Outputs
- Weighted plasmid graph derived from `pling` distances
- Node centrality rankings per metric
- Community structure and bridge-node identification
- Candidate backbone nodes connecting major groups
- Exportable tables/plots for downstream interpretation

---

## Interpretation Guide
- High degree/eigenvector: likely community core nodes
- High betweenness: likely inter-community bridge nodes
- High closeness: nodes with high structural accessibility
- Backbone signal: small set of high-centrality nodes sustaining global connectivity
